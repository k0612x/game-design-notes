## 世界观与故事设定

**时间**：2220年  
**地点**：X1092星（虚拟世界）  
**核心概念**：人类社会中被压制的「自我意识」被剥离后，投放到蝗灾环境中接受「驯化」，直至磨灭自由意志才能被回收重新植入宿主。

### 核心设定

- **主角**：被某回收公司投放的「废弃意识」，任务是捕获蝗虫以换取生存资源
- **世界结构**：不同部落代表不同回收公司，主角所在公司风格放养，信奉「生存耐力极值才能驯化意识」
- **基础规则**：
  - ① 每天选择一项失灵的感官
  - ② 唯一食物来源：每日打卡领小米粥；想吃饱需在黑市卖蝗虫
  - ③ 夜间无安保，参与夜间捕蝗军团可换取物品保管

### 核心情节弧

1. 主角在捕蝗过程中意识到自身超出程序的自主意识
2. 建立抵抗组织，招募野外无人问津的流浪意识
3. 集体回到人类社会与宿主谈判，却发现人类生活更加异化
4. 被回收公司欺骗，自我意识的自我意识再次被剥离，化为漂浮蝗虫景观（参考 Bruce Sterling《Swarm》）

### 荒诞结局之一

> 主角因夜间研究蝗虫亮了大灯，被恐怖分子误认为正规军击毙。

## 故事线迭代记录

### v1 — 初始概念（12月21日）

基于TRPG框架，加入三条时间线设计：

- **过去**：梦境形式呈现（音乐节、蝗灾入侵的记忆）
- **现在**：对「家」的组装装修（玩家选择影响后续剧情）
- **未来**：资源竞争与生存期盼

**感官融入设计：**

| 感官 | 设计方式 |
|------|---------|
| 听觉 | 主角童年音乐节记忆 → 蝗灾摧毁音乐场景，歌手罹患抑郁 |
| 视觉 | 族群用颜色记录日期（周一黄色/周二蓝色）；主角用饲养蝗虫的生命周期纪年（如「蝗虫黄色A一世」） |
| 颜色语义 | 颜色作为形容词使用，赋予独立语义（如回答「绿色」代表某种态度） |

### v2 — 故事线扩展（12月23-30日）

- 主角从「个体捕蝗者」发展为「抵抗组织创立者」
- 加入宿主谈判线、数据库控制线
- 结局反转：解放后的意识空壳反成蝗虫景观，形成叙事闭环

### 玩法参考

- 《辐射》群体设置 —— 但去除架空组织，改为基于生活直觉的命名逻辑
- 参考《银河系搭车指南》的荒诞名词构建风格（如「泛银河含漱爆破液」= 酒）
- 玩家可偷听蝗虫对话，用于提升捕蝗成功率
- 世界无善恶之分，只有点数计算

## 游戏设计思路

### 核心机制

**装修系统（现在时间线）影响后续：**
```
玩家放置蝗虫饲养箱 → 未来战斗中获得额外道具
玩家忽视修复防御  → 未来资源竞争受到威胁
```

**时间线联动逻辑（伪代码）：**
```gdscript
bool hasLocustBox = false

# 玩家在家放置饲养箱时触发
func on_place_locust_box():
    hasLocustBox = true

# 未来场景检测
func check_future_event():
    if hasLocustBox:
        unlock_extra_dialogue()
```

## Godot 技术笔记

### 碰撞系统

- `Area2D` 可添加碰撞形状，用于**检测**（不产生物理推力）
- 物理碰撞需要两个物体的 **layer层** 与 **mask层** 对应
  - `layer`：物体所在层级
  - `mask`：决定哪些对象会与本物体发生碰撞（筛选机制）
- 示例：player（layer 1）碰撞 coin（mask 2）→ player的mask需包含层级2

## 生产流程参考

### 美术/CG生产管线概览

| 阶段 | 工具 | 说明 |
|------|------|------|
| 概念设计 | Photoshop / Procreate | 原画、分镜、风格设定 |
| 3D建模 | Blender / Maya | 高模雕刻 → 低模拓扑 → UV |
| 材质贴图 | Substance Painter | PBR流程（金属度/粗糙度） |
| 绑定动画 | Maya HumanIK | 骨骼绑定 + 关键帧/Mocap |
| 实时渲染 | Godot / Unreal | 游戏引擎内渲染 |
| 后期合成 | After Effects | 过场动画 |

### 项目协作工具

- **ShotGrid**：任务分配、版本追踪、审阅反馈（制作中枢）
- **RV**：高分辨率序列帧审阅，支持多版本对比
- **Confluence**：团队文档库，存储美术规范与技术文档

### 2周快速入门策略

1. 用Blender完整走一遍：建模 → 贴图 → 渲染
2. 在ShotGrid中创建对应任务节点，练习版本追踪
3. 掌握核心快捷键（ShotGrid：空格全局搜索 / RV：F全屏）
4. 向团队索要现成的色彩配置文件，避免从零配置
