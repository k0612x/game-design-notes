## 01 — Kickoff Guide 启动指南

在正式开始 GDD 之前需要明确的核心问题：

| 问题 | 说明 |
|------|------|
| 这是一款什么游戏？ | 用一句话描述核心体验 |
| 目标玩家是谁？ | 受众定位 |
| 核心情感目标 | 玩家玩完后应该感受到什么？ |
| 参考游戏 | 机制参考、叙事参考、美术参考分开列 |
| 规模限制 | 场景数、玩法机制数、主线时长上限 |
| 技术选型 | 引擎、平台、团队规模 |

## 02 — Gameplay Design 玩法设计

### 核心玩法循环

```
[触发事件] → [玩家决策] → [即时反馈] → [状态改变] → [新的触发事件]
```

### 玩家动词（Player Verbs）

列出玩家在游戏中能做的所有动作：

- 移动 / 探索
- 对话 / 选择
- 收集 / 管理资源
- 观察（偷听蝗虫对话等）

### 难度曲线

| 阶段 | 挑战类型 | 设计意图 |
|------|---------|---------|
| 开场 | 低压力，引导探索 | 建立世界观认知 |
| 中段 | 资源压力上升 | 迫使玩家做出取舍 |
| 高潮 | 叙事与玩法同时施压 | 强化情感共鸣 |
| 结局 | 多结局分支 | 反思玩家行为 |

## 03 — Game Mechanics 游戏机制

### 核心机制列表

| 机制名称 | 描述 | 优先级 |
|---------|------|--------|
| 感官失灵系统 | 每天选择一项感官失效，影响操作与信息获取 | 高 |
| 资源经济 | 蝗虫捕获 → 黑市交易 → 生存资源 | 高 |
| 颜色语义系统 | 颜色作为日期记录与沟通方式 | 中 |
| 时间线装修 | 现在的布置选择影响未来剧情状态 | 中 |
| 偷听系统 | 监听蝗虫对话获取行动信息 | 低（迭代） |

### 数值设计原则

- 无善恶值，只有「点数」计算
- 集体捕猎收益 > 个体捕猎（逼迫社交合作）
- 资源稀缺是常态，不是惩罚

## 04 — Story & Narrative Design 叙事设计

### 叙事结构

**三幕式 + 循环叙事**

```
序章（梦境/过去）→ 第一幕（个体生存）→ 第二幕（组织建立）
→ 第三幕（社会回归）→ 尾声（意识再剥离 / 叙事闭环）
```

### 叙事技巧参考

| 技巧 | 来源 | 应用场景 |
|------|------|---------|
| 荒诞命名法 | 《银河系搭车指南》 | 世界观名词构建 |
| 通感修辞 | 原创设计 | 颜色=形容词，感官替代表达 |
| 环境叙事 | Outlast / Amnesia | 通过场景细节传递背景信息 |
| 选择后果延迟显现 | Disco Elysium | 装修选择在未来时间线体现 |

### 对话设计原则

- 避免冗长的名词解释，用行为和语境传递世界规则
- 颜色词汇具有独立语义，不作常规色彩使用
- NPC对话应反映「集体压迫个体」的世界逻辑

## 05 — Worldbuilding 世界构建

### 世界规则摘要

- 资源竞争：人与蝗虫争夺生存资源
- 时间感模糊：族群用颜色着装替代日历
- 部落制：不同回收公司 = 不同意识驯化策略
- 道德体系缺席：只有效率与点数

### 参考资料

- Bruce Sterling《Swarm》— 漂浮物与群体意识
- 《辐射》系列 — 群体结构（去除架空组织，保留生活直觉逻辑）

## 06 — Characters 角色设计

### 主角

| 属性 | 设定 |
|------|------|
| 身份 | 被回收公司投放的废弃自我意识 |
| 核心驱动 | 在规训中维持自主性 |
| 标志性行为 | 用饲养蝗虫的生命周期纪年（如「蝗虫黄色A一世」） |
| 背景记忆 | 游牧族群成员，曾经历17万人音乐节中的蝗灾入侵 |

## 07 — Level Design 关卡设计

### 首版场景规划（控制在3个以内）

| 场景 | 类型 | 核心事件 |
|------|------|---------|
| 蝗虫基地 | 主活动场景 | 资源收集、社交、夜间威胁 |
| 野外 | 探索场景 | 寻找流浪意识、自由探索 |
| 梦境 | 叙事场景 | 过去记忆，解锁现实线索 |

---

## 08 — UI & UX Design

### 设计原则

- UI风格与世界观一致（荒诞、颗粒感、低技术感）
- 颜色系统贯穿UI（日期颜色 / 语义颜色统一）
- 感官失灵状态在UI层面有对应视觉反馈

## 09 — Launch Checklist 发布清单

- [ ] 功能测试，修复关键bug
- [ ] 性能测试（内存、FPS、加载速度）
- [ ] 用户体验测试，收集反馈
- [ ] 符合各平台审查要求
- [ ] 打包并发布（Steam / itch.io）
- [ ] 制定宣传计划（trailer、社交媒体）
- [ ] 软件著作权申请
